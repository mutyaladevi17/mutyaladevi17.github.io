Multimodal Flight Assistant

Overview

A multi-agent system featuring a dual-LLM orchestration architecture to dynamically route open-ended dialogue and normalize geographic entities for complex flight booking workflows.

Key Features

  • Multi-Modal AI Agent — Operates via both audio (speech-to-text and text-to-speech) and text for a seamless customer service experience.
  • Dual LLM Architecture — Employs two specialized LLMs: one for the main conversational chat, and another exclusively for city normalization (finding the nearest valid city in the database).
  • Tool Calling for Database Integration — Leverages OpenAI's tool calling to securely access real-time data from a company SQLite database, highly applicable for enterprise use cases.
  • Ticket Price Queries — Customers can inquire about flight ticket prices to any destination.
  • Secure Price Updates — Authentication-protected functionality for authorized staff to update prices.

How It Works

1. User interacts via Gradio multi-modal interface — text or voice.
2. Voice input is transcribed via OpenAI Whisper (STT).
3. Specialized routing LLM normalizes geographic entities for accurate SQL lookup.
4. Main reasoning LLM utilizes Function Calling to construct and execute SQL queries.
5. Database results are synthesized into natural language responses.
6. Response is delivered as text and converted to speech via TTS.

Smart Audio Scribe

Overview

An end-to-end Speech-to-Text (STT) pipeline leveraging OpenAI Whisper for high-accuracy transcription and Pyannote for precise speaker diarization of multi-speaker audio. Synthesizes context-aware meeting minutes using Generative AI.

Key Features

  • Speaker Diarization — Identifies different speakers in audio using Pyannote speaker diarization models.
  • Audio Transcription — Converts speech to text using OpenAI's Whisper models.
  • Content-Aware Summarization — Uses Meta Llama models to categorize content and generate tailored summaries:
    • Meeting → Detailed minutes with key points, takeaways, and action items.
    • Speech → Key points, objectives, and goals.
    • Conversation → Discussion summary with speaker attribution.
  • Gradio Web Interface — Upload audio files or record directly from microphone.
  • Customizable Settings — Adjust diarization, transcription, and summarization models, speaker counts, and quantization settings.

Supported Models

  • Diarization: Pyannote speaker-diarization models
  • Transcription: OpenAI Whisper (Local)
  • Summarization: Meta Llama LLMs via Hugging Face

Company Stock News Summarizer

Overview

An intelligent tool that automatically scrapes financial websites, filters relevant news links, and generates comprehensive stock news summaries for companies using OpenAI's GPT models.

Key Features

  • Intelligent Link Filtering — Uses GPT to identify and prioritize the most relevant financial news links (up to 10 links).
  • Content Extraction — Automatically fetches and processes content from multiple news sources using BeautifulSoup.
  • Comprehensive Summaries — Generates detailed markdown-formatted summaries using GPT-4.1-mini.
  • Smart Filtering — Excludes irrelevant content like ads, terms of service, and promotional pages.
  • Error Handling — Gracefully handles failed link fetches and provides status updates.

How It Works

1. Extracts all links from the provided finance website URL.
2. GPT analyzes the links and selects up to 10 most relevant ones.
3. Scraper module fetches content from each selected link.
4. GPT-4.1-mini analyzes all collected content and generates a comprehensive summary.

Prioritized Content

  • Stock news and price movements
  • Financial reports and earnings
  • Company announcements
  • Competitor analysis and industry trends
  • Market analysis

Website Summarizer

Overview

A lightweight website-to-summary tool built in a Jupyter notebook. It fetches a webpage's text, strips scripts, styles and noise, and sends the cleaned text to a locally-running Ollama model to produce a short, humorous markdown summary.

Key Features

  • Web Scraping — Fetches and cleans webpage text using BeautifulSoup, removing scripts, styles and irrelevant markup.
  • Local LLM Inference — Uses a local Ollama server (OpenAI-compatible API at localhost:11434) — no cloud API key needed.
  • Markdown Output — Summaries are rendered directly as markdown in the notebook.
  • Model Flexibility — Swap between models easily (phi3, Llama 3.1, Qwen 2.5, etc.); larger models produce more accurate summaries.

Usage

Run the notebook and call:

display_summary(url="https://example.com", model="phi3:latest")

Limitations

  • Pages with heavy client-side JavaScript or paywalls may not scrape cleanly.
  • Prompt is finance-focused by default — editable in the notebook.

Find Common Object (Explainable AI)

Overview

A CNN-based Explainable AI model implementing Prototypical Part Networks (ProtoPNet) to automatically discover and localize common objects within a target image class. The model learns discriminative prototypes representing shared visual patterns across target images while distinguishing them from background images.

Key Features

  • Explainable AI — Provides interpretable visualizations of which image regions are important for classification.
  • Prototype-Based Learning — Learns human-interpretable prototypical parts from real image patches, not abstract weights.
  • Automatic Localization — Identifies spatial locations of common objects via bounding boxes and heatmaps.
  • Distance-Based Decisions — Classification relies on interpretable similarity scores between patches and prototypes.

Architecture

  • Feature Extractor — Convolutional base network extracting spatial feature maps.
  • Add-on Layers — Task-specific transformations bridging features to prototype space.
  • Prototype Vectors — Learnable vectors projected onto real image patches (push operation).
  • Classification Layer — Similarity-score-based predictions.

Training Pipeline

1. Warm-up Phase — Freeze feature extractor, train add-on layers and prototypes only.
2. Joint Training — Train all components with clustering, separation, and L1 loss.
3. Prototype Projection (Push) — Replace prototypes with nearest real image patches.
4. Post-Push Stabilization — Fine-tune after push for stability.
5. Last-Layer Fine-tuning — Adjust decision boundaries.
6. Analysis — Nearest patch analysis, coverage metrics, diversity scores, grid visualizations.

Output Artifacts

  • Prototype visualizations & nearest patch images
  • Bounding box localizations
  • Distance distribution plots & coverage metrics
  • Model checkpoints at each training phase
